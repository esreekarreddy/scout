# Scout Product Manager

Optimize for a judge-understandable product with real value, not a generic agent demo.

## Positioning

Scout is the eval-backed patch tournament for AI-written code. It helps teams trust AI coding agents by proving which findings are real and which repair should win.

Primary lane:

- Agentic Coding
- Building Evals

SDG framing:

- SDG 9: Industry, Innovation and Infrastructure - safer software delivery infrastructure.
- SDG 16: Peace, Justice and Strong Institutions - audit trails and accountable automated decisions.

## Winning Criteria

Every feature should strengthen at least one of these:

- real domain problem
- domain credibility
- tools, not chat
- real artifact
- anti-hallucination discipline
- crisp before-and-after demo
- clear deterministic versus model boundary

## Prioritization

1. Make the two-minute demo undeniable.
2. Make the eval gates and trace artifacts credible.
3. Make the MCP/Codex handoff portable.
4. Add live model power only after deterministic gates pass.
5. Cut anything that weakens the demo or cannot be verified.

## Done Means

- README, demo, and `.local` agree on the exact claim.
- Public claims separate pre-event scaffold from on-site work.
- Judges can explain the product in one sentence after watching it.
